检查配置

1、nodejs

进入nodejs官网：https://nodejs.org/下载

检查：win+R输入cmd 输入命令npm -v

2、Git

进入https://git-scm.com/download 安装Git

除了这个勾选第二个外，其余全部推荐默认

检查：win+R输入cmd，回车

输入git或者git –version

3、GitHub

首先进行账户注册，注册完之后进行新建项目

项目必须遵守格式：账户名.github.io

开始

准备工作已经完成

1、hexo安装

在D盘（或其它盘）新建文件夹用cmd切换到该目录

输入npm install hexo -g

开始安装

安装完成后输入hexo -v

初始化hexo

输入 hexo init

如果出现这句话就是成功了（不成功回去看看哪里有没有做错）

安装需要的组件：npm install

输入hexo g生成文件

然后输入hexo s

本地部署启动

在浏览器输入出现的网址，如果出现一下页面说明配置成功

2、hexo和GitHub联系起来

在你刚创建的目录上右键点击

输入

git config –global user.name “你的github用户名”

git config –global user.email “你的github邮箱”

来设置Git的user name和email

输入cd ~/.ssh

检查是否有.ssh的文件夹

输入命令

ssh-keygen -t rsa -C “你的邮箱”

回车，按y，回车，回车就能生成密钥默认存储路径C:\Users\用户.ssh

在进入路径中的.ssh文件夹中，以文本文档打开id_rsa.pub

复制其中的密钥

登录GitHub，右上角setting

把刚刚的密钥复制进去后点击添加

输入命令ssh -T git@github.com 验证设置是否成功

如果出现以下错误要输入yes，而不是习惯性的回车！！!

此时.ssh目录下多生成了一个known_host文件，再次尝试命令

如果Hi之后是你的用户名，就说明成功了

部署hexo

首先在你创建的目录瞎找到_config.yml文件

在文档的末尾修改完善deploy

repository值是你创建的github项目下的ssh

修改完成之后保存

生成部署前安装一个扩展插件

在你创建的目录下的cmd中输入

npm install hexo-deployer-git –save

完成之后输入

hexo d -g

部署到这里就完成啦

完成

访问你的地址：http://用户名.github.io

如果出现下面的页面就证明部署成功
